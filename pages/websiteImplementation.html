Webseitenimplementierung

Einleitung

Für die Webseitenimplementierung wurde klassisches HTML (Hypertext Markup Language), CSS (Cascading Stylesheet) und die Programmiersprache JavaScript verwendet Es wurde nicht für mobile Endgeräte optimiert. Die Ordnerstruktur ist wie folgt aufgebaut:

  • images
  • pages
  • README.md
  • index.html
  • script.js
  • style.css

Die ersten beiden einträge in der liste, images und pages, sind Ordner. Images beinhaltet die Bilder, die auf der Webseite verwendet werden. Die Bilder sind nicht verlink, da dies die IP-Adresse des Webseitenbesuchers, ohne dessen Zustimmung, an Dritte weiterleiten würde, um die Bilder zu laden. Aus diesem Grund sind alle verwendeten Bilder im images Ordner enthalten.

Im Ordner pages beinhaltet die aktuellen Webseiten. Diese sind klassische HTML dateien. Die Datei index.html ist nicht in diesem Ordner enthalten. Die index.html Datei dient als Eintrittspunkt für die anderen Webseiten. Github pages schaut nach dem Eintrittspunkt in der höchsten Level der Ordnerstruktur. Damit die Webseite erfolgreich veröffentlicht werden kann, muss die index.html Datei sich in dieser Eben der Ordnerstruktur befinden.

Die README.md Datei ist ein Markdown file. Markdown Dateien sind einfache Textdateien, die für formatierten Inhalte verwendet werden. Diese werden oft für Dokumentationen verwendet, da sie mathematische Formeln als auch Code gut darstellen können.

Die Datei script.js enthält javascript code, die auf dem Computer des Nutzer der Webseite läuft. Dadurch können komplexe funktionen implementiert werden, jedoch sind sie auch eine Sicherheitslücke, da diese willkürlichen Code ausführen können

die Datei style.css enthält, wie der Name schon andeuten lässt, den "Style" der Webseite. Zum Beispiel kann hier die Textgröße oder Textfarbe eingestellt werden.

Github pages

m Github pages verwenden zu können, wird ein Github Account benötigt. Wenn dieser vorhanden ist, kann man durch die Github pages statische Webseiten auf github hosten1. Dieser Service ist kostenlose. Wenn nur ein standard Account zur Verfügung steht, dann ist muss die Repository (Repo), die für die Webseite verwendet werden soll, public (öffentlich einsehbar, bedeutet nicht das Dritte auf diese Repo Dateien hochladen können) sein. Mit einem Enterprise oder Organisationen Account können auch private Repos verwendet werden.

Diese Webseite ist unter den Github Repo Kurokousei.github.io zu finden.

Diese Repo beinhaltet alles was diese Webseite ausmacht. Eine Erklärung zu der Ordnerstruktur finden Sie in der Einleitung.

Wenn keine Repo bereits vorhanden ist dann muss eine neue erstellt werden. Um ein neuen Repo zu erstellen, klicke Sie oben rechts auf den + Button und dann auf New Repository.

Dies öffnet eine Webseite, um die Repo zu erstellen und sieht aus wie in Abb. 3.

Füllen Sie das Feld für die Repo mit einem beliebigen Namen aus. Bei einem standard Account wählen Sie für die Sichtbarkeit "Public" aus. Fügen Sie die README hinzu und erstellen Sie die REPO.

Nun sollte ein leeres Github Repo existieren womit Sie eine Webseite auf Github pages hosten können.

Sie können nun oben auf den Settings tab klicken. Dies führt Sie zu den Settings, sieh Abb. 4.

Dort können Sie auf den Pages tab auf der linken Seite klicken. So gelangen Sie auf die Github pages Settings tab.

Oben auf der Abb. 5, können Sie sehen ob die Webseite derzeit veröffentlicht ist. Durch den roten "unveröffentlichen" Button kann die Webseite wieder Offline genommen werden. Damit eine Webseite von Github pages veröffentlicht wird, muss entweder auf der Repo was verändert worden sein, oder es wird unter Branch von der aktuellen Repo auf ein anderes gewechselt (z.B. zu None). Es kann bis zu 10 Minuten dauern, bis die Veränderungen auf der Webseite zu sehen sind.

für eine valide HTML Datei. Tag haben einen opening Tag und einen closing Tag. Es gibt ausnahmen für diese Regel wie für den link Tag.

Im head Tag werden Metadaten und Ressourcen angegeben, diese sind nicht auf der Webseite sichtbar. 


                <!DOCTYPE html>
                <html lang="en" xmlns="">
                <head>
                    <meta charset="UTF-8">
                    <meta name="viewport"
                          content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">
                    <meta http-equiv="X-UA-Compatible" content="ie=edge">
                    <link rel="stylesheet" href="style.css">
                    <script defer src="script.js"></script>
                    <title>Urban Gardening</title>
                </head>

Dies kann zumeinst durch eine integrierte Programmierumgebung (IDE), die HTML unterstützt, automatisch generiert werden. Das meisten ist "Boilerplate" Code. Wichtig hier sind die link und script Tags. Der link Tag verknüpft den Stylesheet mit der HTML Datei. Somit kann in der CSS-Datei der Style des HTML geändert werden. Es linkt nicht zu anderen Webseiten. Der script Tag verknüpft eine Javascript-Datei mit der HTML Datei. Somit kann diese Programme auf dem Computer des Nutzers laufen lassen. Das Keyword defer ist hier wichtig. Es sorgt dafür, dass die Javascript-Datei erst nach der HTML Datei läd. Wenn dies nicht der Fall ist kann es zu problemen führen, also ist meistens eine Gute Idee defer zu verwenden.

Der nav tag an sich bringt keine Funktionalität. Es hat haber semantische Bedeutung. Screen Reader erkennen dies. Somit is es immer gut solche Tags mit semantischer Bedeutung zu verwenden, wenn es Sinn macht, anstatt einem div. Div Tags sind einfache Behälter. In div Tags können andere tags genistet werden. Über CSS und JS Dateien kann auf die div Tags zugegriffen werden. Dies ermöglichst es zum Beispiel die Textgröße alle Tags im div Tag enthalten sind zu ändern. CSS

CSS steht für Cascading Stylesheet. Es ist zuständig für die Presentation und dem Layout der Webseite. In der CSS Datei werden verschiedene Tags, Klassen oder IDs angesprochen, um deren Style oder Layout zu ändern. Folgen ein Beispiel: 


                nav p {
                    margin: 0;
                    color: var(--text-color);
                    font-size: 1.5rem;
                    font-weight: bold;
                }

Hier sind die tags nav und p angegeben. Dies spricht alle p tags im nav Behälter an und ändert ihren Style. Hier werden die Schriftgröße und die Farbe geändert. Wenn der nav Tag fehlen würde, würden alle p Tags auf der Webseite angesprochen werden. Wenn verschiedene solche Codeblöcke existieren, kann es dazu kommen das sie die gleichen Tags, Klassen oder IDs ansprechen. Dann wird nach den Cascading-Regeln entschieden, welches der Codeblöcke letztendlich die Änderung durchführt.

Ein Codeblock für Klassen sieht wie folgt aus: 


                .content-of-table-of-contents {
                background-color: var(--md-sys-color-surface-container-high);
                border-radius: 16px;
                padding: 15px;
                border: 1px solid var(--md-sys-color-outline-variant);
                max-width: 100%;
                box-sizing: border-box;
                }

In CSS können varibalen definiert werden. Dies sieht wie folgt aus: 


                --background: var(--md-sys-color-background);
                --text-color: var(--md-sys-color-primary-fixed-dim);

Die -- sind verpflichtend. Damit werden die CSS variablen als custom variables erkannt. Ohne werden sie nur als einen normalen Eigenschaftsnamen behandelt, was zu fehlern führt. Dies erlaubt es nur eine Variable abzuändern, damit alle die die gleiche Variable verwenden, mit abgeändert werden.

Javascript

Mit JavaScript lässt sich für die Webseite zusätzliche Funktionalität programmieren. Die JS Datei wird von Nutzer beim besuchen der Webseite heruntergeladen. Das Programm läuft dann auf dem Computer des Nutzers. Da diese Dateien willkürliche Programme auf dem Computer des Nutzers laufen lassen können, kann es ein Sicherheitsrisiko für den Nutzer darstellen.

Javascript wird in diesen Projekt dafür verwendet, einen Button sichtbar zu machen, wenn der user 200 px gescrollt hat. Dieser Button scrollt zurück zum Seitenanfang. Die zweite Funktinalität ist das Togglen vom Light mode zum Dark mode und vice versa. Die Preferenze wir im Browser des Nutzers gespeichert, beim aufrufen der Webseite gelesen und dementsprechend gesetzt. 


                const scrollBtn = document.getElementById('scroll-to-top');
                // Show button when scrolled down
                window.addEventListener('scroll', function() {
                    if (window.scrollY > 200) {
                        scrollBtn.classList.add('visible');
                    } else {
                        scrollBtn.classList.remove('visible');
                    }
                });

Dieser Codeblock checked ob der Nutzer 200 px runtergescrolled hat. Wenn ja dann wird der Button sichtbar gemacht. In der ersten Zeile wird der Button in einer konstanten veriable gespeichert. Durch die ID des Buttons steht fest, welche Button angesprochen werden sollen. 


                <button id="scroll-to-top" class="scroll-top-btn" title="Back to top">↑</button>

Dieser button Tag ist auf jeder Webseite über dem closing body Tag. Die ID ist scroll-to-top. Die getElememtById() function findet so welcher Button sichtbar gemacht werden soll. Es wird ein EventListener verwendet, um zu checken, ob der Nutzer beim scrollen insgesamt mehr als 200 px gescrollt hat. Wenn ja dann ist der Button sichtbar. Unter 200 px bleibt der Button unsichtbar.
